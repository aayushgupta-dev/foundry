# 10 — API Specification & Contracts

Base path: `/api/v1`. Auth: `Authorization: Bearer <access_jwt>`; refresh via httpOnly cookie.
Content type: `application/json`; errors: `application/problem+json` (RFC 7807). Full OpenAPI is
generated by FastAPI at `/api/v1/openapi.json` and Swagger UI at `/api/v1/docs`.

## Conventions
- **Pagination (cursor):** `?limit=<n>&cursor=<opaque>` → `{ "data": [...], "next_cursor": "..." }`.
- **Filtering/sorting:** `?filter[status]=indexed&sort=-created_at`.
- **Idempotency:** `Idempotency-Key: <uuid>` on POST/PUT that create/trigger; replay returns the original response.
- **Errors:** `{ "type","title","status","code","detail","request_id" }`.
- **Rate limits:** `429` with `Retry-After`; headers `X-RateLimit-Remaining`, `-Reset`.
- **Versioning:** additive in `/v1`; breaking → `/v2`.

## Auth
| Method | Path | Body / notes | Response |
|---|---|---|---|
| POST | `/auth/signup` | `{email,password,org_name}` — creates org + owner | `201` user + tokens |
| POST | `/auth/login` | `{email,password}` | `200` access token + sets refresh cookie |
| POST | `/auth/refresh` | (refresh cookie) | `200` new access + rotated refresh |
| POST | `/auth/logout` | — | `204`, revokes refresh family |
| POST | `/auth/verify-email` | `{token}` | `200` |
| POST | `/auth/request-verify` | — | `202` |
| POST | `/auth/forgot-password` | `{email}` | `202` (no user enumeration) |
| POST | `/auth/reset-password` | `{token,password}` | `200` |
| POST | `/auth/invites/accept` | `{token,password,full_name}` | `201` |
| GET | `/auth/me` | — | `200` principal + memberships |

## Documents & uploads
| Method | Path | Notes |
|---|---|---|
| POST | `/projects/{id}/uploads` | `{filename,size,mime}` → presigned multipart: `{upload_id, parts:[{part_no,url}], s3_key}`. Idempotent. |
| POST | `/uploads/{uploadId}/complete` | `{parts:[{part_no,etag}], content_hash}` → dedup check; creates document_version; starts ingestion workflow → `{document_id, ingestion_run_id}` |
| POST | `/uploads/{uploadId}/abort` | cancel upload |
| GET | `/projects/{id}/documents` | list (filter by status/kind/tag/folder; sort; paginate) |
| GET | `/documents/{id}` | detail incl. current version + status |
| GET | `/documents/{id}/versions` | version history |
| POST | `/documents/{id}/reupload` | new version (presigned flow) |
| PATCH | `/documents/{id}` | rename / move / tag |
| DELETE | `/documents/{id}` | soft-delete (trash) |
| POST | `/documents/{id}/restore` | from trash |
| DELETE | `/documents/{id}/purge` | hard-delete (S3 + vectors) |
| GET | `/documents/{id}/source-url` | short-lived signed download URL |

## Chat
| Method | Path | Notes |
|---|---|---|
| GET/POST | `/projects/{id}/conversations` | list / create thread |
| GET | `/conversations/{id}` | thread + messages |
| DELETE | `/conversations/{id}` | delete thread |
| POST (SSE) | `/conversations/{id}/messages` | `{content}` → streams assistant tokens, tool steps, citations. `Idempotency-Key` supported. |
| GET | `/conversations/{id}/messages/{msgId}/citations` | resolved citation locators |

**SSE event frames (chat):**
```
event: token        data: {"delta":"..."}
event: tool_call    data: {"tool":"sql_query","args":{...}}
event: tool_result  data: {"tool":"sql_query","rows":[...], "sql":"SELECT ..."}
event: citation     data: {"n":1,"chunk_id":"...","locator":{"page":4}}
event: done         data: {"message_id":"...","usage":{...}}
event: error        data: {"code":"...","detail":"..."}
```
